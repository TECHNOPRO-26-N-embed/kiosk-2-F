## 2. 要求仕様書（顧客ヒアリング結果）

### 2.1 顧客の要望（ヒアリングメモ）

> ここに、顧客（講師）へのヒアリングで得た要望を箇条書きで記録してください。

* 料金→4泊5日 50円 7泊8日 100円
* 延滞料金 +1日 50円
* 返却方法は営業時間内のみ
* ID発行は紙カード
* 貸出時、本の番号/コードを手入力
* 貸出状態のステータス参照
* 破損の可能性を加味して、人間が受け付けて除外処理をする
* ID発行の際、氏名・電話番号・メールアドレス・住所を登録
* 支払方法、現金・クレジットカードのみ
* CSVで出力
* 操作ログの表示
*

## 3. 基本設計

### 3.1 画面遷移図

> テキスト形式で画面の状態遷移を記述してください。
> 例: `メインメニュー → [1] → 購入画面 → [確定] → 完了画面 → メインメニュー`

```
メインメニュー　→　貸出画面　→　本コード入力　→　貸出期間設定　→　確認画面　→　ID入力　→　支払画面　→　完了画面　→メインメニュー
               → 確認画面または支払画面　→　キャンセル　→　一つ状態を戻す
               →　検索機能　→　本コードを入力　→　書籍のおいてある場所を表示
               →　ID登録　→　個人情報登録　→　IDカード発行　→　完了画面　→　メインメニュー

```
