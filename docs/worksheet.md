## 2. 要求仕様書（顧客ヒアリング結果）

### 2.1 顧客の要望（ヒアリングメモ）

> ここに、顧客（講師）へのヒアリングで得た要望を箇条書きで記録してください。

* 料金→4泊5日 50円 7泊8日 100円
* 延滞料金 +1日 50円
* 返却方法は営業時間内のみ
* ID発行は紙カード
* 貸出時、本の番号/コードを手入力
* 貸出状態のステータス参照
* 破損の可能性を加味して、人間が受け付けて除外処理をする

* 支払方法、現金・クレジットカードのみ
* CSVで出力
* 操作ログの表示

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: （書籍検索機能：アルゴリズム日本語ver）

```
1.文字入力画面を表示して書籍名を手入力させる
↓
2.「書籍名」受け取る
↓
3.データベースから「書籍名」で検索する
↓
4.①一文字でも検索がヒットしたら該当書籍のリストを表示する　②検索がヒットしなかったら「該当なし」を表示する
↓
5.①のときはメインメニューor貸出機能に進むかのボタンを表示　②のときは[1.文字入力画面を表示]のページに戻る
```

#### 機能2: （貸出機能）

```
０．メインメニュー
　　貸出ボタン→１へ

１．ID入力
　　if(ID一致)→２へ
　   （ID不一致）→１へ

２．本コードを入力
　　if(ID一致)→３へ
　   （ID不一致）→２へ

３．期間の設定
　　４泊５日又は７泊8日を選択→４へ

４．確認画面
　　貸出期間は〇泊〇日で間違いないですか？
　　if(YES)→５へ
　   （NO）→３へ

５．支払い画面
　　現金又はカード情報を受け取る→６へ

６．完了画面
　　メインメニューへ
```

#### 機能3: （機能名）

```
1.支払画面を表示する
2.貸出料金と延滞料金を表示する
3.合計金額を表示する
4.支払方法を選択
-現金
-クレジットカード
5.入力内容が正しいか確認する
6.現金の場合
-投入金額を確認する
-不足金額を表示
-お釣りを計算する
-支払処理をする
7.クレジットカードの場合
-カード情報を確認する
-支払処理をする
8支払完了場面を表示
9支払内容、貸出内容をCSVに記録する
10.操作ログを記録する
11.完了画面を表示
```

#### 機能4: （利用者ID登録機能）

```
１：貸出状況の情報を取得
２：状況の変更、未貸出/貸出中/延滞中/停止中など
３：貸出状況を保存
４：次の機能へ
５：１or３で情報の取得が失敗した場合処理を中断し前の画面(機能)に戻る
```

#### 機能5: （利用者ID登録機能）

```
1. 規約情報を表示
「読みましたか（1.はい　/2. いいえ）」

2. 「はい」なら4に進む

3. 「いいえ」ならメインメニューに戻る

4. 氏名、電話番号、住所、生年月日、メールアドレスを入力する

5. 入力で記載した内容の確認画面を表示する
-「入力した内容に間違いはないですか？（1,はい /2.訂正）」

6. 「はい」なら→ CSV保存処理を呼び出す → カードを発行していますと表示 →カードを発行しましたと表示

7. 「訂正」なら4に戻る
```
